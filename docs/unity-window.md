# Unity 配表窗口 5 分钟说明

这篇是给策划、主程、项目负责人一起看的日常使用说明。你不需要先理解工具内部怎么实现，只要记住这几条：

- 飞书在线表是正式配表。
- Unity 项目里的本地文件只是给游戏运行和提交 PR 用的缓存。
- 带 `预览` 的按钮都是安全的：只看计划，不写飞书，也不改本地文件。
- `写入`、`创建`、`写回` 会改东西，所以一定会要求你确认。
- 不知道下一步点什么时，先点 `预览同步计划`。

## 打开后先看哪里

打开 `Tools > Config Sheet Forge` 后，先看首页顶部的 `推荐下一步`。

通常你只需要看三件事：

1. 当前状态是不是正常。
2. 推荐你下一步点哪个按钮。
3. 按钮下面写的是不是“只读取，不写文件”。

如果你只是想确认现在有没有问题，点 `预览同步计划`。它不会改任何东西。

## 策划改表流程

你改完飞书在线表后，按这个流程走：

1. 回到 Unity，打开 `Tools > Config Sheet Forge`。
2. 点 `预览同步计划`。
3. 如果预览通过，再到 `配表` 页勾选确认。
4. 点 `写入本地 cache`。
5. 提交 PR，或让主程 / 配置负责人帮你合并。

这里的 `cache` 可以理解成“Unity 项目里的本地配表缓存”。它不是正式源头，正式源头还是飞书在线表。

## 新建配表流程

需要新增一张表时：

1. 进入 `配表` 页。
2. 展开 `新建配表`。
3. 填 `配表ID` 和 `显示名称`。
4. 选择“这张表由谁负责”。
5. 在字段表格里逐行填字段。
6. 点 `预览新建配表`。
7. 找配置负责人确认命名、字段和负责人。
8. 确认无误后，再点 `创建在线表并登记`。

`创建在线表并登记` 会真的写飞书和项目登记信息，所以它会要求你勾选确认，并弹出二次确认。

### 新建配表怎么填

`配表ID` 是代码和文件用的英文 ID，只能用英文、数字、下划线，而且不能以数字开头。比如 `SkillExtraData`。

`显示名称` 是给人看的中文名，比如 `技能扩展数据`。

“这张表由谁负责”选日常维护这张表的人或角色。下面的审批规则是只读的：谁能审字段变化、谁能临时放行、谁能写回 main，仍然由项目规则决定。

字段不用手写模板。每一行填：

- `key`：英文字段名，比如 `id`、`name`、`sortOrder`。
- `中文名`：策划看的字段名。
- `类型`：普通视图里是文本、整数、小数、是/否、日期、日期时间、枚举、JSON。
- `说明`：必填，写清楚这个字段做什么。

第一行默认是唯一 ID 字段，不能删除，但 key 可以改名。至少要保留一个唯一 ID 字段。

如果选择 `枚举`，需要填写枚举值，比如 `common`、`rare`、`legendary`。这样以后同步时，工具能发现“表里出现了未声明的枚举值”。

本地 Excel cache 路径会自动推导，普通新表不用手填。迁移旧 Excel 请去 `本地 Excel Seed`，不要在新建配表页手动填旧文件路径。

## 合并 PR 前

主程或配置负责人合并 PR 前，一般按这个流程：

1. 进入 `合并` 页。
2. 点 `生成合并预览`。
3. 看有没有冲突、字段变更、缺少审查记录。
4. 进入 `PR 检查` 页，点页面顶部的检查按钮。
5. 检查通过后再合 PR。

如果要 `确认写回 main`，这是高风险操作，只应该由负责人确认。

### 目标分支怎么选

如果本机安装并登录了 GitHub CLI，合并页会尝试自动识别当前 PR，并使用 PR 的目标分支。你会看到类似“已识别 GitHub PR #480，目标分支来自 PR：main”。

如果没有安装或没有登录 GitHub CLI，窗口会直接告诉你原因。这时不用慌，仍然可以在合并页搜索目标分支。一般目标分支是 `main`，除非 PR 本身明确指向别的分支。

GitHub CLI 不是同步和新建配表的硬要求。它只让 PR 合并页少一步手动选择。

## 哪些按钮安全

| 按钮 | 会不会改东西 | 适合谁点 |
| --- | --- | --- |
| 刷新状态 | 不会 | 所有人 |
| 预览同步计划 | 不会 | 所有人 |
| 预览新建配表 | 不会 | 策划、配置负责人 |
| 生成合并预览 | 不会写 main | 主程、配置负责人 |
| 运行 PR 检查 | 只生成检查报告 | 主程、配置负责人 |
| 写入本地 cache | 会更新本地缓存 | 确认预览通过后再点 |
| 创建在线表并登记 | 会写飞书和登记信息 | 配置负责人确认后再点 |
| 本地 Excel Seed | 可能迁移旧 Excel | 只给迁移负责人用 |
| 确认写回 main | 会影响主线工作区 | 只给负责人用 |

## 常见提示怎么看

| 你看到的提示 | 现在该做什么 |
| --- | --- |
| 当前分支没有在线表记录 | 先点 `预览同步计划`。如果这是新分支，确认它是否已经接到飞书表；如果是旧 Excel 迁移，找配置负责人处理 `本地 Excel Seed`。Seed 可以理解成“把旧 Excel 第一次迁到飞书”。 |
| 本地 cache 待同步 | 先点 `预览同步计划`，通过后再确认 `写入本地 cache`。 |
| 缺少合并审查记录 | 去 `合并` 页生成合并预览，然后找配置负责人补审查记录。 |
| 字段或表结构审查未完成 | 说明表头、字段或结构可能变了。找配置负责人确认。 |
| 权限不足 | 当前账号或机器人没有权限读飞书表、Wiki 或登记表。找配置负责人开权限。 |
| 临时放行记录过期或无效 | 这是负责人用来临时放行的记录失效了。找负责人更新或重新审查。 |
| GitHub PR 识别：未安装 gh | 仍可手动搜索目标分支。想自动识别 PR 时，安装 GitHub CLI 并登录。 |
| GitHub PR 识别：gh 未登录 | 仍可手动搜索目标分支。想自动识别 PR 时，让主程协助完成 GitHub CLI 登录。 |

## 你不需要看的东西

平时不用看 `输出` 页，也不用展开 `高级诊断`。

只有遇到失败、需要复制错误给工具维护者、或主程排查问题时，再打开它们。

窗口顶部有 `高级模式` 开关。普通使用不用开。打开后会显示内部 key、canonical 类型、路径和完整命令，方便主程或工具维护者排查。

## 给配置负责人看的补充

下面这段普通策划可以跳过。

Unity 顶部 `教程` 菜单会优先使用项目配置里的文档入口。项目可以在 `ProjectSettings/*ConfigSheetForge*.json` 中声明：

```json
{
  "documentationTargets": {
    "5 分钟入门": "docs/tooling/config-sheet-source-of-truth.md",
    "飞书项目配置表": "https://example.feishu.cn/wiki/xxxxx"
  },
  "localDocs": [
    "docs/tooling/designer-config-flow.md"
  ],
  "feishuRootUrl": "https://example.feishu.cn/wiki/xxxxx"
}
```

如果项目没有声明这些入口，窗口会打开 config-sheet-forge 的通用教程和 README。
