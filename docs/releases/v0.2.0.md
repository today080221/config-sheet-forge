# v0.2.0 发布说明

## 亮点

- 新增类型行优先的 semantic import，支持 `string`、`number`、`integer`、`bool`、`date`、`datetime`、`json`、`enum`。
- Lark provider 支持 bot 优先、user fallback，并增强 `data/items/records/values` 等 JSON shape 解析。
- `gate` 支持 `--annotations github`，可在 GitHub Actions 中输出 PR annotation。
- Unity 包新增 edit-mode tests，并抽出可测试的 Editor helper。
- 本地 CI 扩展到 build、测试、Unity 包结构、私有内容扫描和 CLI pack；公开 GitHub Actions 因缺少 `workflow` scope 暂未恢复。
- 项目文档改为中文维护。

## 迁移说明

- v0.1 的首行字段名导入仍可工作。
- 推荐新表显式配置 `--field-row 0 --type-row 1 --description-row 2 --data-start-row 3`。
- `providerSettings.larkCliIdentity` 默认是 `bot`，provider 会在 bot 操作失败时 fallback 到 user。
- `lark-cli sheets +export` 的输出路径要求在工作目录内使用相对路径，v0.2.0 provider 已处理。

## 验证摘要

- `pwsh scripts/Run-CI.ps1` 通过，包含 build、12 个测试、Unity 包结构检查和私有内容扫描。
- `dotnet pack src/cli/ConfigSheetForge.Cli/ConfigSheetForge.Cli.csproj -c Release` 已生成 `ConfigSheetForge.Cli.0.2.0.nupkg`。

## 已知限制

- `json` 类型当前做对象/数组边界和平衡校验，不做完整 JSON canonical rewrite。
- GitHub annotation 暂以 semantic 文件为定位文件，表格行列上下文保留在 finding location 中。
- Lark docs search 的可用性取决于当前 bot/user scopes；配置好的 root 仍会作为候选 root 展示。
